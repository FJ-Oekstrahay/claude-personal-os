Follow the OpenClaw governance model for deploying a change. Before doing anything:
1. Read `~/.openclaw/workspace/GOVERNANCE.md`
2. Describe the task as a task envelope: what, which agent, evidence required, rollback plan
3. Dry run: describe exactly what will change, what the side effects are, what could go wrong
4. Stop and wait for Geoff's explicit "go ahead" before executing anything
5. After execution: provide evidence (command output, file diffs) per GOVERNANCE.md rule 1
