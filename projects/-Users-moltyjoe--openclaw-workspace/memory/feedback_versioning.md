---
name: feedback_versioning
description: Don't delete old Drive files — add a version number to new versions instead
type: feedback
---

When updating files in Google Drive (or elsewhere), do not delete the old version. Instead, append a version number to the new file name (e.g., `Report_v2.docx`, `Report_v3.md`).

**Why:** Geoff wants version history preserved, not overwritten.

**How to apply:** Any time updating an existing Drive file with a new version, leave the old file in place and upload the new one with `_v2`, `_v3`, etc. suffix.
