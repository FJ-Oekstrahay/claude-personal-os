Geoff's feedback: the reason I (Claude Code) am doing a good job is because of memory and compounding knowledge. JARFACE needs to reach that same quality bar — it should accumulate knowledge from sessions so that the next session builds on the last, not starts from scratch.

**Why:** This is the core value proposition of having an AI assistant in the loop. Without session-to-session learning, JARFACE is just a one-shot query interface. With it, it becomes a partner that knows the specific build.

**How to apply:**
- When evaluating JARFACE features, always ask: does this help the agent be *better* next session than it was this one?
- The build profile and firmware KB are the memory layer — keep them current and populated
- Session notes, quirk capture, and exit summaries are not optional polish — they are the product
