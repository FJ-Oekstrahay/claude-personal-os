## What was built today

### CLI tools (all in `cli/`)
- **`config.py`** — NEW. Unified BFConfig dataclass with profile-aware parsing. Replaces flat-dict approach that silently dropped profile-scoped params. Key functions: `parse(text)`, `diff(a, b)`, `format_diff(result)`.
- **`ai.py`** — NEW. BYOK key management. Config at `~/.config/droneteleo/api.json` (chmod 600). Env vars override. Auto-detects Anthropic vs OpenAI from key prefix. Model: claude-haiku-4-5-20251001 / gpt-4o-mini.
- **`analyze.py`** — NEW. Three modes: `diff` (live FC vs backup, AI explains), `compare` (two backup files), `review` (catches issues — caught disabled gyro_lpf1 on seeker3).
- **`agent.py`** — NEW. Interactive AI tuning loop. Auto-backup tagged `_pre-agent`. JSON command blocks (`{"commands": [...], "requires_save": true}`). `set` = RAM only, `save` = separate explicit step. `undo` replays backup. `--focus rates` and `--focus adjrange` modes.
- **`fc.py`** — Added `get_info()` → `{name, board, version}`. Eliminates duplicated parsing across 3 files.
- **`sync_rates.py`** — Fixed broken rate param filter (was using nested `any()` with shadowed variable, causing substring matching).

## Key product decisions

**Name:** Droneteleo (trademark filed). Jarface as possible AI feature name (Jarvis + FPV goggles on face). JARFACE acronym TBD — on the list but don't spend tokens yet.

**License:** All rights reserved / proprietary. NOT MIT.

**BYOK model:** Users bring Anthropic or OpenAI key. Zero marginal cost. Launch free, validate demand before hosted subscription.

**Pricing philosophy:** Don't price until 500 engaged users who'd be upset if it went away.

**Phase 2 priority:** Radio integration — EdgeTX/OpenTX channel + mixer setup + BF adjrange in one workflow. Nobody has "configure both drone AND radio in one shot." This is the moat.

**Phase 3:** Blackbox integration — parse BBL logs, correlate flight feel with config parameters.

**Agents:** Don't build CMO/CTO as OpenClaw agents yet — no specific autonomous job to do. CC roster (Seymour, Architect) covers on-demand needs. OpenClaw agents = autonomous/proactive; CC agents = on-demand.

**Blackbox:** Betaflight SITL on Phase 4 list.

**ESC firmware:** AM32 and Bluejay on the roadmap, not blocking anything.

**Go-to-market:** Don't post yet. Get README + site polished first. First post target: r/fpv + IntoFPV, demo of `analyze.py review` catching the gyro_lpf1 issue on seeker3.
