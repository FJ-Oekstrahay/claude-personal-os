## Key decisions made

**GUI architecture:** Serial port proxy (socat + Python bridge) is the target v1 architecture. BF stays connected to virtual port throughout; pilot watches tabs update in real time. Critical unknown: does Chrome WebSerial accept socat pty devices? Spike code written (cli/proxy.py). Hardware test pending.

**BF 10.10.0:** Under evaluation as bundled GUI. Uses chrome.serial (not WebSerial) — fully CDP-automatable without permission dialogs. GPL-3.0 licensed. Test without hardware is possible.

**Playwright automation:** Blocked by WebSerial requestPort() dialog (OS-level, not automatable). Only useful for Disconnect click; Connect step still requires user interaction. Falls back to v1.5 role if proxy works.

**Why:** Tabs disappear on BF disconnect — no live GUI without proxy. This was the key insight that changed the architecture priority.
