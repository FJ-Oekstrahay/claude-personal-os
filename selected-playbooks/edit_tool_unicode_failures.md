## Problem

The Edit tool's `old_string` matching fails (returns "string not found") when the target text contains:
- Em dashes: `—` (U+2014)
- Curly/smart quotes: `"` `"` `'` `'`
- Arrows: `→`
- Other non-ASCII Unicode

This is especially common in markdown docs written by hand or generated by Claude (which uses typographic punctuation).

## Fix

Use Python `str.replace()` via Bash:

```bash
python3 - <<'EOF'
with open('/path/to/file.md', 'r') as f:
    content = f.read()

# Paste exact old text directly (copy from Read tool output)
old = '- [ ] **Item** — Description with em dash'
new = '- [x] **Item** — Updated description'

if old in content:
    content = content.replace(old, new, 1)
    with open('/path/to/file.md', 'w') as f:
        f.write(content)
    print("OK")
else:
    print("NOT FOUND")
    # Debug: find nearby text
    idx = content.find("Item")
    if idx >= 0:
        print(repr(content[idx:idx+100]))
EOF
```

If still not matching, use `repr()` to inspect the exact bytes:
```bash
python3 -c "
with open('file.md') as f: c = f.read()
idx = c.find('keyword near target')
print(repr(c[idx:idx+200]))
"
```

## When to use

Any time Edit fails with "string not found" on a markdown file that was hand-written or contains typographic punctuation. Check first with a targeted Grep; if the text exists but Edit fails, reach for Python.
